## Implemented

### 1) Dual-cam TV page on camera server
- Added endpoint: `/tv`
- File: `naval_web_control/scripts/naval_cam_server.py`
- Purpose: full-screen friendly page with LEFT and RIGHT camera feeds and FPS/status.

### 2) HDMI kiosk launcher script
- Added file: `naval_web_control/scripts/naval_hdmi_kiosk.sh`
- Behavior:
  - waits for camera server health (`/health`)
  - detects Chromium
  - checks display session availability (`DISPLAY` / `WAYLAND_DISPLAY`)
  - launches fullscreen kiosk to `http://127.0.0.1:8090/tv?kiosk=1`
  - safely exits without error if no GUI/display session is available

### 3) Non-breaking start script integration
- Updated file: `naval_web_control/start_naval.sh`
- Added optional HDMI attempt after camera server starts.
- Added flag: `--no-hdmi` (disable HDMI kiosk per run).
- Existing ROS2 launch flow unchanged.

### 4) User systemd service template
- Added file: `naval_web_control/systemd/naval-hdmi-kiosk.service`
- Purpose: optional autostart of HDMI kiosk in user session.

### 5) Deployment documentation
- Added file: `naval_web_control/HDMI_KIOSK_SETUP.md`
- Includes remote headless rollout steps via SSH/Tailscale:
  - install `xorg openbox lightdm chromium-browser curl`
  - force HDMI in `/boot/firmware/config.txt`
  - configure LightDM autologin
